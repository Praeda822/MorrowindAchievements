Register New Category

  1. Add the following code to your main.lua file:
    local interop = require("sb_achievements.interop")
        
local Category = interop.registerCategory("Cat Name")

Register New Achievement

  1. Add the following code to your main.lua file:
    local interop = require("sb_achievements.interop")
        
--- @param e initializedEventData
local function initializedCallback(e)
    interop.registerAchievement {
        id         = "ach_1",
        category   = "Main",
        condition  =
            function()
                return tes3.getJournalIndex { id = "A1_1_FindSpymaster" } >= 1
            end,
        icon       = "Icons\\mod_folder\\icn_ach_1.tga",
        colour     = interop.colours.yellow,
        title      = "Achievement Name",
        desc       = "Achievement description."
        configDesc = interop.configDesc.hideDesc
        lockedDesc = "Achievement not unlocked."
    }
end
event.register(tes3.event.initialized, initializedCallback)
  2. The following parameters are required:
    • id = the unique ID of the achievement
    • category = the category of the achievement:
      • Register new categories like above, or use an existing category fom another mod.
    • condition = the condition that unlocks the achievement:
      • This should be a function that returns a boolean.
    • icon - the icon path:
      • The icon should be a DDS or TGA type.
    • fullIcon - the alternative icon path:
      • The icon should be a DDS or TGA type.
        Note: This parameter can be used if you wish to fully replace the background of the default icon.
    • title - the title of the achievement
    • desc - the description of the achievement
  3. The following parameters are optional:
    • colour - the icon background colour:
      • The default colours are red, yellow, green, blue, indigo, violet, white, and black.
      • Alternatively, a new colour can be assigned in the format 
        {r / 255, g / 255, b / 255}
        .
    • configDesc - the description display rules for the achievement, overriding the user's MCM settings:
      • hideDesc - hides the description until the achievement is unlocked
      • showDesc - shows the description of the achievement even when locked
      • groupHidden - hides the achievement entirely until it is unlocked
    • lockedDesc - the description of hte achievement when locked:
      • This defaults to an empty string when not given a value and configDesc is set to hideDesc.
      • Under lockedMessage I created template messages for steamDetailsRevealed, steamKeepPlaying, xboxSecret, xboxMoreYouPlay, and psHidden.
